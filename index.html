02 — The On Switch
/loop
Give it a task and a timer. It runs on repeat until you stop it.
/loop 5m check if the build passed and tell me the result /loop 10m look for new GitHub issues and summarize them /loop 2m run the tests and fix anything that fails
To stop: say end the loop — or write the exit condition into the prompt itself.

04 — Build Your Own Tools
Custom Commands
One markdown file = one new slash command. Build a CLI that knows your workflow.
Create: ~/.claude/commands/standup.md Run: /standup

06 — Event-Driven Guardrails
Hooks
Catch the habits you keep forgetting you have.
// settings.json "hooks": { "PostToolUse": [{ "matcher": "Bash", "hooks": [{ "command": "if echo $INPUT | grep -q 'git commit'; then echo 'Run your tests!'; fi" }] }] }
After a commit → remind. After a file edit → format. Before a push → lint.
No plugins. No extensions. Ralph is the brain. Hooks are the guardrails.
